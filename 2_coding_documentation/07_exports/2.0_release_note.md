# 2.0 Release Note（课堂演示版）

更新时间：2026-04-21  
项目：小饭桌（大学生做饭学习规划师）

## 1）2.0 版本定位
- 面向课堂展示与作业提交的**桌面网页演示版**
- 目标：清晰展示“学习规划闭环”与“mock-only / RAG-ready 架构”

## 2）相比 1.0 增加了什么
- 桌面端页面壳：左侧导航（高亮/徽标占位）+ 顶部条（非首页）
- 首页 2.0：Hero 输入主卡 + 快捷胶囊 + 右侧辅助区（随机/贴士/省钱计划）
- 新增结果页：loading/空/错/成功列表集中在结果页；支持返回改条件与重新生成
- 详情页 PRD2：食材/调料点选高亮、badge 数字、加入待购反馈弹窗、响应式双栏
- 待购清单：按菜谱分组展示，详情页支持新增/编辑/删除/勾选（桌面端左右分栏）
- 收藏页：搜索+筛选胶囊+网格展示；支持导出 JSON（本地下载）
- 最近做过：详情页“我做了”写入记录，最近页不再永久空
- 新手贴士：技巧卡片网格（mock）
- 体验增强：toast/skeleton/modal 动效、成功提示音素材位、随机弹窗一次性彩带动效
- AI core（占位）：Settings 模型配置（localStorage）；provider 抽象（mock-only）
- RAG-ready（占位）：本地 JSON 知识库检索 → promptBuilder 组装（仍走 mock 输出）

## 3）哪些地方仍是 mock / 占位
- 不接真实 LLM / OpenClaw / 云端 provider（仅保留接口与 TODO）
- 不接真实后端/数据库/云端知识库
- RAG 只做本地 JSON + 关键词检索（结构 ready，不是完整 RAG）
- 省钱计划弹窗为 mock 展示（无真实计算/后端）

## 4）已知限制
- HashRouter：线上 URL 需要带 `/#/`（例如 `/#/shopping`）
- 成功提示音可能受浏览器自动播放策略影响（需先用户点击；替换后可能要硬刷新）
- 结果页不持久化：刷新 `#/results` 可能回到 idle 提示（不影响收藏/清单/最近做过）

## 5）适合课堂演示的亮点（建议强调）
- 结果页中间层：让首页更聚焦输入与约束，候选对比更清晰
- 学习闭环完整：详情页选择→加入待购→清单 CRUD；收藏/最近做过沉淀
- mock-only 但架构完整：params/prompt/retrieval/provider 抽象清晰，后续替换真实服务成本低
- 体验增强可控：动效/音效都不绑架主逻辑，可替换/可降级
