## 1）目录总览（你需要导出什么）

### 核心过程文档（建议转 PDF）
- `00_project_brief/`：范围边界、rubric 映射、执行手册（如老师要看“项目依据/规则”）
- `04_issue_reviews/`（v1）+ `08_v2_upgrade/03_issue_reviews_v2/`（v2）：问题、风险与反思（课堂复盘/总结用）
- `08_v2_upgrade/01_prompt_logs_v2/`：v2 每轮 prompt 归档（证明“vibe coding”过程）
- `08_v2_upgrade/02_step_reports_v2/`：v2 每轮 step report（证明“方案→落地→验证”）
- `08_v2_upgrade/04_flowcharts_v2/`：v2 流程图（从 v1 到 v2 的推进与收口）

### 版本与发布记录（建议保留 md + 可选转 PDF）
- `03_git_records/commit_index.md`：提交/版本记录（含无法提交时的命令建议）
- `03_git_records/release_notes.md`：简要发布记录（如需要）

### 截图/素材（建议保留原始文件夹）
- `08_v2_upgrade/05_assets_v2/`：每轮 before/after 截图、素材记录

### 最终导出（本目录，建议转 PDF）
- `07_exports/`：本次“最终提交/演示”汇总文档（索引/流程概览/分工模板/演示脚本/最终 checklist/release note/大纲）

## 2）推荐导出顺序（最省时间）
1. `07_exports/vibe_coding_master_outline.md`（总大纲）
2. `07_exports/vibe_process_overview.md`（过程概览）
3. `07_exports/demo_script.md`（课堂演示脚本）

## 3）哪些文件用于“PDF”，哪些用于“项目记录”
- **适合 PDF**：`07_exports/`、`08_v2_upgrade/02_step_reports_v2/`（挑关键几轮）、`08_v2_upgrade/03_issue_reviews_v2/`、`04_flowcharts_v2/`
- **适合保留为 md**：`commit_index.md`、`final_deploy_report.md`（便于复制命令/链接）
- **建议作为附件保留**：`05_assets_v2/` 截图目录、`3_presentation/`（演示脚本/打包 zip）

## 4）最终交付物位置提示
- 代码：`1_project_files/`
- 文档：`2_coding_documentation/`
- 演示材料：`3_presentation/`
- 证书/凭证：`4_certs/`（如老师要求上传证书、队员信息、声明等）
