# 课堂演示脚本（v2 最终版）

更新时间：2026-04-21  
目标：3–5 分钟讲清楚“做饭学习规划”的闭环 + mock AI/RAG-ready 架构 + vibe coding 增量过程。

## 0）开场（10 秒）
一句话：这是“小饭桌 2.0”演示版——**纯前端、本地 mock**，不接真实 AI/后端，但架构已 RAG-ready，适合课堂 MVP 展示。

## 1）启动与访问（给现场同学/老师）
```powershell
cd 1_project_files/cooking-ai-planner
npm.cmd install
npm.cmd run dev
```
打开：`http://localhost:5173/#/`

## 2）演示主路径（推荐顺序）

### Step A：首页（输入 → 约束 → 生成）
1. 指向 Hero：**“今天想吃点什么？”**
2. 输入一句话（例）：
   - “预算 10 元内，20 分钟，宿舍小锅，想吃清爽的”
3. 点 1–2 个快捷胶囊（如“20 分钟内”“<10 元”“仅宿舍小锅”）
4. 点击主 CTA：**生成菜谱**
5. 解释：
   - 当前生成来自 mock provider（可控、稳定、适合课堂）
   - 但会把输入/筛选/检索结果组织成统一 params → prompt（后续可替换真实模型）

### Step B：结果页（候选对比 → 进入详情）
1. 展示“菜谱结果”与**条件摘要**
2. 看到 3–5 张候选卡片（网格）
3. 点击任意卡片进入详情页
4. 兜底：
   - 结果为空/报错：点“重新生成”或“返回修改条件”

### Step C：详情页（学习闭环：选择 → 加入待购 → 反馈）
1. 快速扫：标题/主图/标签/时长/预算/难度
2. 在“食材/调料”区域点选 2–3 项（按钮右上角出现 badge 数字）
3. 点击“加入待购清单”→ 出现结果反馈弹窗
4. 点击“查看待购清单”

### Step D：待购清单（分组 + CRUD）
1. 首页按菜谱分组展示卡片（数量/进度）
2. 点进任意分组进入清单详情（桌面端左右分栏）
3. 点右下角 FAB “+”新增 1 条；演示勾选/编辑/删除
4. 强调：状态在 localStorage，刷新仍保留

### Step E：收藏与最近做过（沉淀页的价值）
1. 回到详情页点击“收藏”→ 去“我的收藏”看到数量变化
2. 回到详情页点击“我做了”→ 去“最近做过”看到记录出现
3. 解释：这是学习规划产品的“沉淀闭环”

## 3）亮点速讲（30 秒）
- **mock-only / RAG-ready**：不接真实 AI，但 params/prompt/retrieval/provider 已抽象，便于替换
- **结果页中间层**：把列表/再生成/返回改条件从首页迁出，首屏更聚焦
- **体验增强但不绑架主逻辑**：toast、skeleton、模态动效、成功提示音、随机彩带（都可替换/可降级）

## 4）降级演示策略（现场网络/权限不稳定时）
- 没网：本项目纯前端，本地可完整演示（不依赖云端）
- 音效没响：浏览器可能拦截音频；先点击任意按钮，再试；或按 `public/sounds/README.md` 硬刷新/清缓存
- 端口被占用：`npm.cmd run dev -- --port 5174`

## 5）必须演示的页面（老师最容易认可）
- 首页 → 结果页 → 详情页（选择→加入待购反馈）
- 待购清单详情 CRUD
- 收藏跨页同步 + “我做了”→最近做过
