# Step 2.0-10 Prompt Log（视觉与布局修正 / P0）

## 输入 prompt 原文（用户）

当前任务是在现有 2.0 网站基础上做第一轮视觉与布局修正，不允许重建项目，不允许推翻现有技术路线。

请严格遵守《执行手册2.0（包含自归档）》中的要求：
- 增量修改
- 每轮只做一个明确步骤
- 同步更新 prompt log、step report、issue log、flowchart、git 记录
- 本轮结束后必须做最小验证（npm run build + 关键页面运行检查）
- 输出必须包含：本轮目标、修改文件清单、实施方案、代码实现、运行与验证、文档归档更新、git 记录、下一轮建议

====================
【本轮目标】
====================
完成 2.0 第一轮页面修正，重点处理以下三个问题：

1. 全站视觉主色重新统一
2. 页面整体主内容区偏窄，需要加宽
3. 首页最核心的 Hero 输入区（“今天想吃点什么？”这一块）需要明显加宽，目标约为当前宽度的 1.5 倍

====================
【必须修改的设计要求】
====================

一、全局色彩系统重定向
请先抽离或统一全局 design tokens / CSS variables / theme constants，然后按以下颜色规范修改：

- 页面主背景色：#faf9f5
- 次要按钮 / 次级浅底块：#f3f3ec
- 重要高亮按钮 / 主强调色：#ff7c24

要求：
1）不要把整站做成大面积橙色
2）整体视觉气质要更克制、更干净、更偏桌面网页产品感
3）橙色只作为唯一主强调色使用
4）以下位置优先改成橙色强调：
   - 主 CTA 按钮
   - 当前导航选中态
   - 数字角标 / badge
   - 重点高亮标签
   - 弹窗主按钮
   - hover / active 的重点交互态
5）以下位置保持克制：
   - 页面大背景
   - 卡片主体底色
   - 正文文字
   - 次级说明文字
   - 大部分边框和分割线

二、整体页面宽度加宽
当前 2.0 页面虽然已经是桌面网页布局，但主内容区仍然偏窄，视觉上还有“从窄屏页面拉宽”的感觉。

请调整全局布局比例：
1）左侧导航宽度可以基本保持稳定
2）中间主内容区需要明显加宽
3）右侧辅助区保留，但不要压缩主内容区
4）整体页面要更像真正的桌面网页，而不是窄屏居中内容

三、首页 Hero 输入区重点加宽
请重点修改首页最重要的输入区域，也就是包含以下内容的模块：
- “今天想吃点什么？”标题
- 说明文案
- 大输入框
- 生成菜谱按钮
- 上传入口
- 快捷筛选区

要求：
1）该 Hero 模块整体宽度提升到当前的大约 1.5 倍
2）让这个区域成为首屏最明显的视觉焦点
3）保证加宽后布局更舒展，不要挤压右侧“随机一道菜 / 学做饭小贴士 / 省钱计划”模块
4）调整后仍保持网页端清晰层级，不要造成元素过分松散
5）当前首页不允许重新引入“今日练习氛围”板块

====================
【实现要求】
====================

1）不要推翻现有 2.0 页面结构，只做增量修正
2）优先修改：
   - AppLayout / 全局容器布局
   - HomePage 布局比例
   - 全局配色 token
   - 按钮、badge、导航当前态、CTA 的颜色
3）如果项目里已经有全局样式文件、主题变量文件、layout 组件，请优先在这些地方改，不要到处硬编码颜色
4）请明确列出：
   - 哪些文件属于文件级改动
   - 哪些属于代码级改动
   - 哪些属于视觉/样式级改动
5）本轮不要改业务逻辑，不要接真实 AI，不要新增复杂功能，只处理视觉与布局修正
6）本轮属于 2.0 调整中的高优先级修正项，请在 step report 中标记为 P0
