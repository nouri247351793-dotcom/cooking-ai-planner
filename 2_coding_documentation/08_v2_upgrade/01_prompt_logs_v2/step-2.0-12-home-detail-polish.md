【必须修改的具体要求】
====================

一、首页模块间距统一（问题 09）
1）统一首页主列所有模块的纵向间距，不要出现某一块特别松、某一块特别挤
2）重点检查：
   - “更多筛选”与“今日灵感”之间
   - 推荐卡片区上下留白
   - 首页各主要卡片块之间的间距
   - 右侧辅助卡片之间的间距
3）主列与右列的模块节奏尽量保持一致
4）尽量抽出统一的 spacing token / layout spacing 规则，而不是零散硬改多个 margin

二、首页底部“今天的灵感”模块移除，推荐区标题改为“今日灵感”（问题 10）
1）移除首页底部独立存在的“今天的灵感”板块
2）将上方推荐卡片区标题统一改为“今日灵感”
3）保留现有推荐/灵感卡片内容，但统一收束到“今日灵感”模块下
4）删除底部板块后，重新调整页面收尾留白，避免出现突兀空白或断层
5）页面底部应更简洁，不再重复表达“灵感”内容

三、背景改为纯色 + 预留浅橙色波点素材位（问题 11）
1）全站背景不要使用渐变
2）统一背景基色为：#faf9f5
3）如果项目中存在渐变背景、叠色背景、模糊渐变层，请清理或替换为纯色
4）允许增加一个浅橙色波点装饰素材位，但要满足：
   - 素材位供开发者后续替换 png / webp / svg
   - 当前没有正式素材时可安全留空
   - 素材位不能压住主内容，不能影响可读性
5）装饰层应该是可选的、轻量的，而不是背景主角
6）不要因为去掉渐变而让页面变得发灰或层级消失，要用卡片阴影、边框和留白维持层次

四、菜谱详情页底部操作栏独立成区（问题 12）
1）当前详情页底部“收藏”和“加入待购清单”按钮区不要继续和上方内容卡片重叠
2）将这组按钮区放到内容卡片下方，作为单独一栏 action section
3）该区域需要与上方“推荐搭配 / 营养信息 / 步骤”等内容保持清晰层级
4）重新处理：
   - 内容区底部间距
   - 按钮栏顶部间距
   - 页面收尾留白
5）保留原有功能：
   - 收藏逻辑
   - 加入待购清单逻辑
   - badge 数字逻辑
6）目标是让详情页底部操作更稳定、更像独立功能区，而不是压在内容卡片上的浮层

五、详情页食材/调料选择区增加“清空”按钮（问题 13）
1）在食材/调料点选加入待购清单的区域增加“清空”按钮
2）“清空”按钮用于清除当前详情页里的临时已选对象，不影响：
   - 已经加入待购清单的数据
   - 收藏状态
   - 页面其他内容
3）点击“清空”后需要实现：
   - 所有当前选中项取消高亮
   - “加入待购清单”按钮右上角数字角标归零并隐藏
4）“清空”按钮位置要清晰，但不能抢主 CTA
5）按钮语义必须明确为“清空当前选择”，避免被理解为“清空整个待购清单”
