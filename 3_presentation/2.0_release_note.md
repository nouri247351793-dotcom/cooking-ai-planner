# “小饭桌”2.0（演示版）发布说明

日期：2026-04-21  
范围：课堂展示 / 交作业演示（纯前端、本地 mock）

## 1）本版本目标
- 用“桌面网页信息架构”呈现做饭学习规划流程
- 形成可演示闭环：输入 → 候选结果 → 详情学习 → 加入待购/收藏/最近做过
- 保持“mock-only / RAG-ready”架构：不接真实 AI、不接真实后端/数据库

## 2）主要更新（2.0 相对 1.0）
- 桌面端壳：左侧固定导航（高亮/徽标占位）+ 顶部条（非首页）
- 首页 2.0：Hero 主输入卡（输入+上传占位+快捷胶囊+更多筛选）+ 右侧辅助区（随机/贴士/省钱计划）
- 新增菜谱结果页：承载 loading/空/错/成功列表，支持返回改条件与重新生成
- 菜谱详情页 PRD2：双栏响应式、食材/调料点选高亮、badge 数量、加入待购结果反馈弹窗
- 待购清单：按菜谱分组卡片；详情页桌面端左右分栏；支持新增/编辑/删除/勾选
- 收藏页：搜索+筛选+网格，收藏跨页同步；预留本地 JSON 导出 service
- 最近做过：通过“我做了”按钮打通闭环，支持空状态与记录展示
- 新手贴士：技巧卡片网格（本地 mock）
- UX-lite：toast、skeleton、轻量 modal 动效、随机弹窗一次性彩带动效、成功提示音素材位
- AI core（占位）：Settings 模型配置（localStorage）；services/ai provider 抽象；本地知识库 + 关键词检索（RAG-ready）

## 3）不包含的内容（明确非目标）
- 不调用真实 LLM / OpenClaw / 云端 Provider
- 不接真实后端/数据库/云端知识库
- 不做统一状态管理大重构（仅 Context + hooks 的轻量数据层）

## 4）运行方式
```bash
cd 1_project_files/cooking-ai-planner
npm.cmd install
npm.cmd run dev
```
访问：`http://localhost:5173/#/`

## 5）已知限制 / 风险
- `HashRouter`：分享链接需要带 `#/`，否则刷新可能不在预期页面。
- 成功提示音：可能受浏览器自动播放策略/站点权限影响；替换音效后可能需要硬刷新清缓存。
- 结果页状态不持久化：刷新 `#/results` 可能回到 idle 提示（不影响收藏/清单/最近做过等持久化数据）。
- Vite build 警告：提示 `optimizeDeps.disabled` 相关迁移（不影响当前 build 产物）。
