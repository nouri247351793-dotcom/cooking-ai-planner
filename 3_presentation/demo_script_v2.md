# “小饭桌”2.0 课堂演示脚本（3–5 分钟）

前置：先在项目根目录启动开发服务器（Windows 推荐用 `npm.cmd`）

```bash
cd 1_project_files/cooking-ai-planner
npm.cmd install
npm.cmd run dev
```

打开：`http://localhost:5173/#/`

---

## 0）开场（10s）
- 介绍：这是“大学生做饭学习规划师（小饭桌）”的 2.0 演示版（纯前端、本地 mock，不接真实 AI/后端）。
- 目标：演示“输入→生成候选→学习详情→加入待购→收藏/最近做过”的学习闭环。

## 1）首页 Hero：输入 + 条件（40–60s）
- 指向大标题“今天想吃点什么？”
- 在大输入框输入一句话：例如“预算 10 元内，20 分钟，宿舍小锅，想吃清爽的”
- 点一两个快捷胶囊（如“20 分钟内”“<10 元”“仅宿舍小锅”）
- 点击主按钮“生成菜谱”
- 讲解：先走 mock 生成链路，但会把 params 组织成统一结构，未来可直接接入 LLM / RAG。

## 2）结果页：候选对比（30–40s）
- 看到“菜谱结果”标题 + 条件摘要
- 展示 3–5 张候选卡片网格
- 点击任意卡片进入详情页
- 兜底：如果结果为空/报错，点“重新生成”或“返回修改条件”

## 3）详情页：学习与操作闭环（60–90s）
- 快速扫：标题/主图/标签/时间/价格/难度
- 在“食材/调料”区域点选 2–3 项（右下角“加入待购清单”出现 badge 数字）
- 点击“加入待购清单”，出现结果弹窗
  - 讲解：同名条目会做合并数量兜底；所有数据暂存在 localStorage
- 点击“查看待购清单”

## 4）待购清单：分组 + CRUD（40–60s）
- 在“待购清单”页看到按菜谱分组的卡片
- 点进某个分组进入清单详情（桌面端左右分栏）
- 点右下角 “+” FAB 新增一条（例如“西红柿 2 个”）
- 勾选完成/编辑/删除任意一条，刷新页面确认持久化（可口头说明）

## 5）收藏与最近做过（40–60s）
- 回到详情页，点“收藏”
- 去“我的收藏”：看到收藏数量 + 搜索/筛选胶囊 + 卡片网格
- 回到详情页，点“我做了”
- 去“最近做过”：看到刚记录的菜谱卡片（按最近排序）

## 6）轻量氛围增强（20–30s）
- 首页右侧点“随机一道菜”：弹出模态层（遮罩+居中偏上）
  - 出现一次性彩带/纸屑动效
  - 可点“再摇一次”“查看详情”
- 点“省钱计划”：弹出“本月省钱计划”弹窗（mock 内容）
- 去“新手贴士”：看到 3 张技巧卡网格

---

## 演示失败兜底（10s）
- 音效没响：浏览器可能拦截音频，先点击页面任意按钮再试；或检查系统音量/站点权限。
- 端口被占用：改端口 `npm.cmd run dev -- --port 5174`
