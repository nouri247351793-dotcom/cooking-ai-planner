# “小饭桌”2.0 最终检查清单（Final QA）

## 1）构建与启动
- [ ] 安装依赖：`cd 1_project_files/cooking-ai-planner` 后执行 `npm.cmd install`
- [ ] 本地开发：`npm.cmd run dev`（如端口占用：`npm.cmd run dev -- --port 5174`）
- [ ] 构建产物：`npm.cmd run build` 成功
- [ ] 访问地址：`http://localhost:5173/#/`

## 2）一级页面可达性（左侧导航）
- [ ] 开始做饭（首页）`#/`
- [ ] 待购清单 `#/shopping`
- [ ] 我的收藏 `#/favorites`
- [ ] 最近做过 `#/recent`
- [ ] 新手贴士 `#/tips`
- [ ] 设置 `#/settings`

## 3）关键路径交互（最小可演示）
### 首页 → 结果页
- [ ] Hero 输入框可输入（提示文案：Enter/Shift+Enter）
- [ ] 快捷胶囊可点击（不会阻塞生成）
- [ ] 点击“生成菜谱”出现加载态，成功后跳转到 `#/results`
- [ ] 成功跳转时触发一次成功提示音（若被浏览器策略拦截，至少不报错）

### 结果页 → 详情页
- [ ] 结果页显示：标题 + 条件摘要 + 3–5 张卡片
- [ ] 点击卡片进入详情页 `#/recipes/:recipeId`
- [ ] “返回修改条件”回到首页且尽量保留输入/筛选
- [ ] “重新生成”可再次触发生成（loading → success/empty/error）

### 详情页（学习与闭环）
- [ ] 基础信息完整：标题/图/标签/时间/价格/难度/食材/调料/步骤/营养/推荐搭配
- [ ] 食材/调料可逐项点选，高亮明确
- [ ] “清空”按钮只清除本页临时选择（不影响已加入清单/收藏）
- [ ] “加入待购清单”按钮 badge 数字与选中数一致
- [ ] 点击“加入待购清单”弹出结果反馈弹窗（可进入待购清单）
- [ ] “收藏”可切换，跨页同步到“我的收藏”
- [ ] “我做了”写入最近做过，跨页同步到“最近做过”

### 待购清单（分组 + 详情 CRUD）
- [ ] 首页按菜谱分组展示卡片：名称/缩略图/物品数/进度
- [ ] 点击分组进入详情页：桌面端左右分栏（左：食材；右：调料+器具）
- [ ] 新增（FAB）/编辑/删除/勾选完成可用
- [ ] 刷新后数据仍在（localStorage 持久化）

### 我的收藏
- [ ] 显示：标题 + 收藏数量 + 搜索 + 筛选胶囊 + 卡片网格
- [ ] 搜索覆盖：菜名/食材/标签
- [ ] 取消收藏后从列表移除（渐隐可有可无，但不应卡死）
- [ ] 收藏跨页同步（详情页收藏 → 收藏页立即变化）

### 最近做过 / 新手贴士
- [ ] 最近做过：无记录时空状态文案清晰；有记录后展示卡片网格
- [ ] 新手贴士：至少 3 张技巧卡，插画/标题/时长/标签完整

### 随机一道菜弹窗
- [ ] 点击“随机一道菜”打开模态：全页暗化、居中偏上、右上角关闭
- [ ] “再摇一次”刷新内容且再次触发一次轻彩带动效
- [ ] “查看详情”进入该菜谱详情页
- [ ] 彩带动效不遮挡按钮（且 `prefers-reduced-motion` 时自动降级）

### 省钱计划弹窗
- [ ] 点击“省钱计划”卡片弹出“本月省钱计划”弹窗
- [ ] 弹窗可关闭（按钮/ESC/遮罩点击如支持）

## 4）localStorage 基线（应保留）

## 5）常见问题处理
- 音效替换：见 `public/sounds/README.md`
- `npm` 被 PowerShell 拦截：用 `npm.cmd` 或调整 ExecutionPolicy
- URL 分享：本项目用 `HashRouter`，分享时需带 `#/`（例如 `http://localhost:5173/#/shopping`）
